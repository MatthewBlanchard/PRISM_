Class Cell

A cell is a single tile on the map.

It defines the properties of the tile and has a few callbacks. Maybe cells should have components so that they can be extended with custom functionality like the grass? Still working on the details there. For now, cells are just a simple way to define the properties of a tile.

Fields

cell.lightReduction Reduces the amount of light that passes through this cell by the integer value in this field.
cell.movePenalty Applies a penalty to speed when moving through this cell by the integer value in this field.
cell.name Displayed in the user interface.
cell.opaque Defines whether a cell can be seen through.
cell.passable Defines whether a cell is passable.
cell.sightLimit If set to an integer, an actor standing on this tile's sight range will be limited to this number.
cell.tile Defines the tile appearance.

cell:visibleFromCell(level, cell)
Determines if a cell is visible from another cell. Cells can have custom functions to determine whether an actor standing on them can be seen. For instance, grass cells allow actors to be seen only if the other actor is in the same clump of grass.

Parameters:

  • level Level The level where the visibility check is taking place.
  • cell Cell The cell from which visibility is being checked.

Returns:

    boolean True if the cell is visible from the provided cell, false otherwise.
