Classes

Action An 'Action' is a command that affects a discrete change in the game state.
Actor An 'Actor' represents entities in the game, including the player, enemies, and items.
ActorStorage The 'ActorStorage' is a container for 'Actors' that maintains a list, spatial map, and component cache.
Cell A 'Cell' is a single tile on the map.
Component The Component class represents a component that can be attached to actors.
Condition A Condition is an event handler that is attached to an actor.
Event A private class that is exclusively instantiated by Condition.
Level The 'Level' holds all of the actors and systems, and runs the game loop.
Reaction The 'Reaction' class represents an action that's performed in response to another action.
Scheduler The 'Scheduler' manages a queue of actors and schedules their actions based on time.
System A 'System' is a class representing a level-wide event handler that can be attached to a Level object.
Target A 'Target' is a discriminator used to validate the target of an action.
Vector2 A Vector2 represents a 2D vector with x and y components.
Object A simple class system for Lua.
